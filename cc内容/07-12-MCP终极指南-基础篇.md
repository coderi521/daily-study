## 🧠 核心概念

### 什么是 MCP？

| 项目 | 说明 |
|------|------|
| 全称 | **Model Context Protocol**（模型上下文协议） |
| 发布 | Anthropic 于 2024年11月25日发布 |
| 本质 | 让大模型能够使用各类工具的**协议标准** |
| 类比 | 相当于给大模型装上了"工具箱" |

> **MCP 的核心价值**：大模型本身只会问答，MCP 让它拥有了使用外部工具的能力（浏览器、编辑器、API 等）。

### MCP 架构三要素

```
┌─────────────────────────────────────────────────┐
│                   MCP Host                       │
│            (支持 MCP 的客户端软件)                  │
│                                                   │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐  │
│  │  LLM API  │ ←→ │ MCP Host │ ←→ │MCP Server│  │
│  │  (模型)   │     │  (调度器) │     │  (工具)   │  │
│  └──────────┘     └──────────┘     └──────────┘  │
└─────────────────────────────────────────────────┘
```

| 组件 | 说明 | 例子 |
|------|------|------|
| **MCP Host** | 支持 MCP 协议的客户端软件 | Cline、Cursor、Cherry Studio、Claude Desktop |
| **MCP Server** | 提供工具的**本地程序**（不是远程服务器！） | weather-server、fetch-server |
| **Tool** | MCP Server 内部的**功能函数** | `get_forecast()`、`fetch_url()` |

> ⚠️ **重要理解**：MCP Server 不是传统意义上的远程服务器，它就是一个本地程序，跟手机 App 没有本质区别。只是它的运行符合 MCP 协议规范。

### Tool 是什么？

- Tool（工具）本质就是编程语言里的**函数**
- 每个 Tool 有：**输入**（参数）→ **处理**（逻辑）→ **输出**（结果）
- 类比：就像手机时钟 App 里的世界时钟、闹钟、计时器、秒表

## 🔧 实操指南

### 1. 安装 MCP Host（以 Cline 为例）

1. 安装 VS Code
2. 在扩展市场搜索 **Cline** 插件并安装
3. 安装后 VS Code 侧边栏会出现 Cline 图标

### 2. 配置 API Key

推荐使用 **Open Router**（聚合多种模型 API）：

| 步骤 | 操作 |
|------|------|
| 1 | 注册 Open Router → 右上角 Sign In |
| 2 | 点击 Keys → Create Key |
| 3 | 填写名称 + 额度上限（可选） |
| 4 | **保存 Key**（页面关闭后无法找回） |
| 5 | 需要付费模型 → 充值 Credits |

**模型选择建议**：

| 模型 | 费用 | 效果 |
|------|------|------|
| Claude 3.7 | 较贵 | 对 MCP 支持最好 |
| DeepSeek V3 | 便宜 | 效果也不错 |
| Open Router 免费模型 | 免费 | 不太稳定 |

在 Cline 中配置：
- API Provider → Open Router
- Key → 刚才保存的 Key
- Model → 选择模型
- **注意**：Plan Mode 和 Act Mode 都要配置

### 3. 配置 MCP Server

MCP Server 的配置方式有两种：

#### 方式 A：自动安装（AI 主导）
- 在聊天中让 Cline 自动创建并安装
- 或者在 MCP Server 市场点击 Install

#### 方式 B：手动配置（推荐，更可控）
点击 Cline 的 Configure MCP Server，编辑 JSON 配置文件：

```json
{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": ["--directory", "/path/to/server", "run", "server.py"],
      "disabled": false,
      "timeout": 60,
      "transportType": "stdio"
    }
  }
}
```

| 配置项 | 说明 |
|--------|------|
| `command` | 启动 MCP Server 的程序（如 `uv`、`npx`） |
| `args` | 启动参数 |
| `disabled` | 是否禁用 |
| `timeout` | 连接超时时间（秒） |
| `transportType` | 通信方式，`stdio`（主流）或 `sse` |

> `stdio` 模式下，MCP Server 通过标准输入/输出与 Host 通信。

### 4. 使用他人的 MCP Server

#### 方式一：uvx（Python 程序）

1. **安装 UV**（Python 包管理工具）：
   ```bash
   # macOS
   curl -LsSf https://astral.sh/uv/install.sh | sh
   ```
2. **验证安装**：
   ```bash
   uvx pycowsay hello world
   ```
3. **配置 MCP Server**：
   ```json
   {
     "mcpServers": {
       "fetch": {
         "command": "uvx",
         "args": ["mcp-server-fetch"]
       }
     }
   }
   ```
4. **解决首次超时问题**：先在终端手动执行 `uvx mcp-server-fetch` 下载依赖，再回到 Cline 重试连接。

#### 方式二：npx（Node.js 程序）

1. **安装 Node.js** → 官网下载安装即可（自带 npx）
2. **配置 MCP Server**：
   ```json
   {
     "mcpServers": {
       "hotnews": {
         "command": "npx",
         "args": ["-y", "@wopal/mcp-server-hotnews"]
       }
     }
   }
   ```

> **uvx vs npx**：uvx 装 Python 程序，npx 装 Node 程序，功能类似

## 🔄 MCP 交互流程详解

这是视频中最核心的部分，理解了这个流程就理解了 MCP 的本质：

### Step 1：注册阶段（MCP Server 启动瞬间）

```
Cline ──"你好，我是 Cline"──→ MCP Server
Cline ←──"你好，我是 weather"── MCP Server
Cline ──"你有什么 Tool 可用？"──→ MCP Server
Cline ←──"我有 get_forecast 和 get_alerts"── MCP Server
```

> ✅ 这一步在 MCP Server 启动时**瞬间完成**

### Step 2：用户提问阶段

用户：*"纽约明天的天气怎么样？"*

```
① Cline 把问题 + 已注册的 Tool 列表 → 发给 LLM 模型

② 模型判断：
   "我的知识截止到去年12月，不知道纽约天气"
   "但 weather server 有 get_forecast 这个 Tool"
   "让我调用它试试"

③ 模型 → Cline："我要调用 get_forecast(40°N, 74°W)"

④ Cline → MCP Server：
   "请调用 get_forecast，参数是 40°N, 74°W"

⑤ MCP Server → 调用天气 API → 拿到结果 → 返回给 Cline

⑥ Cline → 模型："Tool 调用结果拿到了"

⑦ 模型 → 根据结果总结答案 → 返回给 Cline

⑧ Cline → 用户：显示最终答案
```

### 完整流程图示

```
用户 ──问题──→ Cline ──问题+Tool列表──→ LLM 模型
                                           │
                                           │ 判断需调用 Tool
                                           ↓
                   Cline ←──请求调用 Tool── 模型
                     │
                     ↓ 转发调用请求
              MCP Server
                     │
                     ↓ 执行函数
              天气 API (外部)
                     │
                     ↓ 返回数据
              MCP Server
                     │
                     ↓ 返回结果
                   Cline
                     │
                     ↓ 转发结果
                  模型 ──总结答案──→ Cline ──→ 用户
```

## 💡 关键要点总结

| 要点 | 说明 |
|------|------|
| 🏗️ **MCP Server ≠ 远程服务器** | 它就是本地程序，只是遵循 MCP 协议规范 |
| 🔧 **Tool = 函数** | 每个 Tool 就是编程里的一个函数，有输入和输出 |
| 🔄 **交互是"模型主导"** | 模型判断是否需要调用 Tool 以及调用哪个 |
| 📋 **Tool 列表在启动时注册** | MCP Server 启动瞬间完成能力声明 |
| 📦 **uvx / npx 自动管理依赖** | 首次运行需要下载，可能超时，建议先手动执行一次 |
| 💡 **手动配置更可控** | 虽然 AI 能自动安装，但手动配置知道自己在做什么 |
